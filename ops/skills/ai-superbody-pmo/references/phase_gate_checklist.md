# AI Superbody Phase Gate Checklist

## Phase 1 gate
- [ ] Ubuntu host readiness is clear
- [ ] stable stack baseline is installed
- [ ] `simctl run` produces `run_result.json`
- [ ] `report` and `replay` entry are usable

## Phase 2 gate
- [ ] stable closed-loop route is repeatable
- [ ] L0 smoke and L1 regression can batch run
- [ ] KPI gate and failure taxonomy exist
- [ ] digest can be generated from results

## Phase 3 gate
- [ ] public-road asset bundle is standardized
- [ ] at least one replay scenario is in the unified flow
- [ ] Top 5 corner cases are tracked
- [ ] shadow comparison inputs and metrics are documented
