# PIX-Simulation-Validation-Platform

## 超体计划后续事项（云端整理版｜细化执行）

> 适用场景：将“超体计划”后续工作直接迁移到飞书/Notion/Confluence/Jira/TAPD 等云端协作工具。

---

## 0. 云端空间初始化（第 0 天）

### 0.1 目录结构（建议）
- `00_项目管理`：章程、里程碑、周报、会议纪要
- `01_需求管理`：需求池、需求评审、需求基线
- `02_方案设计`：架构设计、接口设计、评审记录
- `03_研发实现`：开发计划、分支规范、发布记录
- `04_仿真与验证`：仿真配置、测试用例、验证报告
- `05_风险与质量`：风险台账、问题单、复盘
- `06_归档`：历史版本与冻结文档（只读）

### 0.2 权限模型（建议）
- 管理员：项目负责人、PM、技术负责人
- 编辑：模块负责人、执行成员
- 只读：外部协作方、旁路支持团队

### 0.3 基础规则（第 0 天必须生效）
- 命名规范：`项目-模块-主题-YYYYMMDD-vX.Y`
- 文档状态：`草稿 / 评审中 / 已发布 / 已归档`
- 变更策略：任何关键文档更新必须记录“变更原因 + 影响范围 + 回滚方式”

---

## 1. 目标与范围对齐（第 1-2 天）

### 1.1 里程碑定义（模板）
| 里程碑 | 时间 | 核心交付物 | 验收标准 |
|---|---|---|---|
| M1 | T+2 周 | 需求基线、初版架构方案 | 需求冻结率 ≥ 90% |
| M2 | T+4 周 | 关键模块可运行版本 | 核心场景通过率 ≥ 85% |
| M3 | T+6 周 | 验证报告与上线建议 | 验证闭环率 ≥ 95% |

### 1.2 范围管理
- In Scope：本阶段必须完成的功能与验证范围。
- Out of Scope：延后项（必须记录“延后原因”与“回收时间点”）。
- 范围变更：由 PM 发起，技术负责人评估，项目负责人审批。

### 1.3 输出物
- 《范围说明 v1.0》
- 《里程碑验收口径》

---

## 2. 任务拆解与负责人分配（第 2-3 天）

### 2.1 工作流拆分
1. 产品流：需求梳理、优先级、验收口径
2. 算法流：模型方案、参数策略、效果评估
3. 仿真流：场景构建、仿真执行、结果沉淀
4. 验证流：用例设计、缺陷跟踪、结论输出
5. 运维流：环境配置、发布策略、监控告警

### 2.2 任务卡片字段（在云端看板统一）
- 标题（动作 + 对象）
- 描述（输入、步骤、输出）
- DRI（直接负责人）
- 协作人
- 优先级（P0/P1/P2）
- 截止时间
- 验收标准（可量化）
- 依赖项与阻塞项

### 2.3 状态流转（建议）
`待开始 -> 进行中 -> 待评审 -> 已完成 -> 已归档`

---

## 3. 数据与文档治理（第 3-5 天）

### 3.1 文档基线清单（必须补齐）
- 需求基线文档
- 系统/模块接口说明
- 仿真配置说明
- 验证报告模板（含结论与追踪项）

### 3.2 版本与追溯
- 每周五固定打版本标签：`weekly-vX.Y`
- 每个关键结论必须可追溯到：输入数据 -> 执行版本 -> 输出结论
- 禁止覆盖历史文件，统一“新版本增量发布”

### 3.3 归档策略
- 关闭 30 天的任务自动进入归档目录
- 归档文档默认只读，修改需申请

---

## 4. 风险与依赖管理（每周持续）

### 4.1 风险台账字段
- 风险编号
- 风险描述
- 风险等级（高/中/低）
- 触发条件
- 影响范围（进度/质量/成本）
- 应对措施（规避/缓解/转移/接受）
- 责任人
- 下次复核时间

### 4.2 常见风险检查清单
- 外部数据源可用性不足
- 第三方服务 SLA 不稳定
- 仿真环境与生产环境差异过大
- 关键人员带宽不足

### 4.3 会议机制
- 每周风险例会（30 分钟）
- 输出《风险周报》并同步处理状态

---

## 5. 研发与验证节奏（固定节拍）

### 5.1 周节奏
- 周一：计划排期与目标确认
- 周三：中期检查（偏差识别 + 纠偏）
- 周五：结果复盘（产出、问题、下周重点）

### 5.2 缺陷 SLA（建议）
| 优先级 | 首响时间 | 解决时限 | 说明 |
|---|---|---|---|
| P0 | 2 小时内 | 24 小时内 | 阻塞主流程 |
| P1 | 8 小时内 | 3 个工作日内 | 影响核心能力 |
| P2 | 1 个工作日内 | 7 个工作日内 | 一般优化项 |

### 5.3 双向追踪
- 每条验证结论需关联：任务卡 + 代码版本 + 仿真记录
- 每个关键缺陷需关联：发现用例 + 修复提交 + 回归结果

## 7. 本周立即执行的 3 件事（升级版）

1. **半天内**完成“里程碑 + 责任人 + 截止日期”三项绑定并发布 v0.1 看板。
2. **1 天内**把现有资料迁移到新目录，按命名规范补齐版本号与状态标记。
3. **本周五前**完成首次风险评审会，形成《风险周报 #1》并明确 Top3 风险负责人。

---

## 8. 验收口径（判断整理是否真正落地）

满足以下 5 项即视为“云端整理完成”：
- 有统一目录与权限模型，并可被全员访问。
- 所有进行中任务均有 DRI、截止时间、验收标准。
- 四类基线文档齐全且可追溯版本。
- 风险台账已建立并至少更新 1 次。
- 周节奏已运行至少 1 个完整周期并形成复盘记录。
