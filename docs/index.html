自动管理

项目自动化如何运行

工作日 digest

GitHub Actions 会在工作日早上生成任务和场景 digest，抓取 overdue、due soon、blocked、场景关注项和最新验证快照。

条件邮件提醒

如果配置了 SMTP Secrets，就自动发给团队；如果没有，则保留 digest artifact、workflow summary 和 digest issue。

GitHub Digest Inbox

即使暂时没有 SMTP，也会自动创建或更新仓库里的 digest issue，作为团队公开提醒入口。

管理规则

P0 延期、阻塞项和新场景必须进入周会；Notion 保持详细计划，GitHub 保持公开执行镜像和自动提醒输入。

算法路线

未来端到端怎么做

推荐路线：BEVFusion + UniAD-style shadow

现有 `BEVFusion` 感知继续作为主基线，`UniAD-style` planner 先并行运行，`VADv2` 作为对照路线，先做公开道路 shadow 评测，再进入固定路线受控闭环。

推进顺序

`BEVFusion` 输出映射 → shadow 指标 → `UniAD-style` 与 `VADv2` 对照 → fallback 策略 → 受控闭环，不直接上端到端控制接管。

当前遗漏

还没有完成但必须盯住的点

真实运行时未落地

控制平面、文档和自动化已齐，但 WSL2、Autoware 工作区、CARLA 0.9.15 与桥接的真闭环还未在当前机器上完成。

场景资产还在起步

公开道路资产束和 Top 5 corner case 的结构已经定了，但还需要把现场问题真正转成可回归场景。

提醒系统未完全闭环

digest 已能自动生成并更新 issue，邮件发送还缺 SMTP Secrets；远端 GPU 节点也仍是明确阻塞项。
