项目周期

完整的 12 周推进方式

1-2

环境与底座

完成 Ubuntu 22.04 主机、ROS 2 Humble、Autoware 工作区和 CARLA 0.9.15 的可重复准备。

3-4

最小闭环与桥接

打通 autoware_carla_interface、时钟、TF、控制反馈和 L0 smoke 报告模板。

5-8

自动化验证

固化 run_result、回放、报告、KPI 门禁和 L1 回归，形成日常可复用验证链路。

9-12

公开道路与研究线

标准化公开道路资产、落地场景与 corner case，并推进 CARLA 0.9.15 上的 E2E shadow 对照。

技术架构

三条主线如何分工

Stable

稳定闭环主线

公司 Ubuntu 主机负责 CARLA 0.9.15、Autoware Universe、ROS 2 Humble 和桥接，目标是形成可重复的最小闭环。

→

Public Road

场景与资产主线

把 lanelet、pointcloud、rosbag、重建输入和现场问题标准化成可复用资产束、场景模板和 corner case。

→

E2E Shadow

研究与对照主线

在同一条 CARLA 0.9.15 运行基线上推进 BEVFusion、UniAD-style 和 VADv2 的 shadow 对照。
