项目周期

完整的 12 周推进方式

1-2

环境与底座

完成公司 Ubuntu 22.04 主机、ROS 2 Humble、Autoware 工作区和 CARLA 0.9.15 的可重复准备。

3-4

最小闭环与桥接

7-8

9-10

公开道路场景落地

至少让 1 个公开道路场景和 1 个高价值 corner case 进入正常验证流程。

11-12

季度收口与下阶段准备

冻结季度验收集，复盘风险与阻塞项，并把 UE5 / E2E shadow 的下阶段入口准备好。

自动管理

项目自动化如何运行

工作日 digest

GitHub Actions 会在工作日早上生成任务和场景 digest，抓取 overdue、due soon、blocked、场景关注项和验证快照。

条件邮件提醒

如果配置了 SMTP Secrets，就自动发给团队；如果没有，则保留 digest artifact、workflow summary 和 digest issue。

GitHub Digest Inbox

即使暂时没有 SMTP，也会自动创建或更新仓库里的 digest issue，作为团队公开提醒入口。

管理规则

P0 延期、阻塞项和新场景必须进入周会。Notion 保持详细计划，GitHub 保持公开执行镜像和自动提醒。
