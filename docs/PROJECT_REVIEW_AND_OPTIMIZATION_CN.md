# 项目复盘与优化建议

## 当前状态

仓库目前已经具备这些基础能力：

- `simctl` 控制平面
- GitHub / Notion 双看板
- digest 自动汇总
- 稳定栈启动骨架
- 公开道路资产与研究场景骨架

但真正决定季度交付的关键仍然是：

- 公司 Ubuntu 主机是否能完成环境搭建
- `CARLA 0.9.15 + Autoware` 是否能形成第一条真实闭环
- `run_result -> report -> replay` 是否能稳定出结果

## 当前最大缺口

1. 真实闭环还没有在目标主机上完成验证
2. 公开道路资产已经有骨架，但还缺首个可重复回归输入
3. 研究线需要继续保留，但必须统一在 `CARLA 0.9.15 / UE4.26` 口径

## 优先级优化

### P0

- Ubuntu 主机访问与权限
- ROS 2 / colcon / rosdep / vcs
- CARLA 0.9.15 / Town01 / recorder
- Autoware 工作区、依赖解析、首次编译
- 第一条 `run_result.json`

### P1

- `site proxy` 资产束
- Top 5 corner case
- E2E shadow 对照指标
- 研究场景回放与报告模板

### P2

- 邮件提醒
- 更复杂的公开道路重建链路
- 更大规模的研究场景矩阵

## 优化建议

- 不再保留独立研究栈，所有研究任务统一落到 `stable`
- 研究线只保留“对当前主线有帮助”的结论，不做孤立展示
- 所有项目管理提醒都围绕季度硬门槛：环境、闭环、数据闭环、资产束
